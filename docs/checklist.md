# 개발 백로그 (작업 목록)

앞으로 할 일을 우선순위 순으로 쌓아둔 목록이다. 주차마다 위에서부터 꺼내 그 주에 끝낸다.
상황이 바뀌면 순서를 다시 매기고, 못 끝낸 건 다음 주로 넘긴다. 설계 단일 소스는 docs/schema.md,
수집 세부는 docs/데이터-수집.md, 전체 맥락은 CLAUDE.md.

## 우선순위

- P0 필수: 없으면 서비스가 성립하지 않는다. 먼저 한다.
- P1 중요: 신뢰와 완성도. P0 다음에 한다.
- P2 나중: 여유되면 한다.

## 4주 로드맵

| 주차 | 목표 |
|---|---|
| 1주차 | 기획, 설계, 디자인, 개발 환경 (완료) |
| 2주차 (이번 주) | 화면(React mock) + 실데이터 파이프라인 첫 삽 (크롤링·LLM 파싱 스파이크) |
| 3주차 | 총망라 (소스 최대한 붙이기, 소스 간 중복 제거, LLM 파싱 안정화) |
| 4주차 | 믿을 수 있게 (정확도 측정, 상세 근거, 다듬기, 배포) |

이번 주(2주차)는 React 화면을 mock으로 만들고(완료), 제일 불확실한 크롤링·LLM 파싱을 먼저 찔러본다. 총망라(소스를 최대한 붙이는 것)는 3주차다. 요일별 상세는 [주간계획.md](주간계획.md).
