# 개발 백로그 (작업 목록)

앞으로 할 일을 우선순위 순으로 쌓아둔 목록이다. 주차마다 위에서부터 꺼내 그 주에 끝낸다.
상황이 바뀌면 순서를 다시 매기고, 못 끝낸 건 다음 주로 넘긴다. 설계 단일 소스는 docs/schema.md,
수집 세부는 docs/데이터-수집.md, 전체 맥락은 CLAUDE.md.

## 우선순위

- P0 필수: 없으면 서비스가 성립하지 않는다. 먼저 한다.
- P1 중요: 신뢰와 완성도. P0 다음에 한다.
- P2 나중: 여유되면 한다.

## 4주 로드맵

| 주차 | 목표 |
|---|---|
| 1주차 | 기획, 설계, 디자인, 개발 환경 (완료) |
| 2주차 (이번 주) | 수직 슬라이스: 화면-서버-DB 한 사이클 (React + Express + Supabase) |
| 3주차 | 데이터 넓히기 (크롤링 소스, 다중 소스 중복 제거, LLM 파싱) |
| 4주차 | 믿을 수 있게 (정확도, 상세 근거, 다듬기, 배포) |

이번 주(2주차)에 반드시 끝낼 것은 화면부터 DB까지 한 바퀴가 실제로 도는 것이다. React 화면에서 조건을 넣으면 Express 서버가 Supabase를 조회해 매칭 결과를 화면으로 돌려주는 한 사이클을, 시드 데이터로라도 완성한다. 데이터를 넓히는 건 다음 주다. 요일별 상세는 [주간계획.md](주간계획.md), 작업은 GitHub 이슈 #1부터 #7로 쪼개 뒀다.
