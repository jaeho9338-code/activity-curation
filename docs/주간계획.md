# 2주차 주간 계획

이번 주는 화면(mock)을 만들고, 실데이터 파이프라인에서 **제일 불확실한 부분(크롤링·LLM 파싱)을 먼저 찔러본다.**
총망라가 핵심이라 소스에서 빠짐없이 긁는 게 목표지만, 그 전에 "한 소스라도 진짜 긁혀서 파싱되나"부터 확인한다.
쉬운 것부터 쌓다가 3주차에 크롤링에서 막히는 걸 피하려는 거다.

## 요일별

### 월 - 계획·이슈·문서 (완료)
주간 계획 세우고 작업을 GitHub 이슈로 쪼갬. 기획·데이터 전략 정리.

### 화 - React 핵심 화면 (완료)
조건 입력 -> 가능/확인 필요/거의 가능 분류 화면을 mock으로. 검색·카테고리 필터·즐겨찾기·정렬·공고 상세 모달·
컴포넌트 분리까지. 서버 없이 mock으로 화면이 도는 걸 확인.

### 수 - 제일 불확실한 것 먼저 찌르기 (스파이크)
- 링커리어 F12 > Network 열어 내부 API가 있는지 확인. 있으면 JSON 몇 건 받아본다.
- 콘테스트코리아 상세 한 건을 Claude에 넣어 자격조건이 제대로 뽑히는지 본다. **여기가 이 프로젝트의 심장이다.**
- 되면 나머지는 다 되고, 안 되면 기획을 일찍 고친다.

### 목 - 실데이터 한 소스 수집 -> 저장
- Supabase 테이블을 단순하게(수집 원문 + 완성본) 만든다.
- 한 소스(링커리어 내부 API 또는 온통청년 API)에서 진짜 공고를 받아 저장. 고유 키로 중복 방지.

### 금 - 화면에 진짜 공고 연결 + 정확도 확인
- mock 대신 저장된 진짜 공고를 화면에 띄운다.
- 공고 몇 건을 손으로 정답 라벨링해 매칭이 맞는지 눈으로 확인.

## 이번 주에 안 하는 것
- 여러 소스 총망라(전량 수집)는 3주차다. 이번 주는 "한 소스라도 진짜 흐르나"까지.
- 매일 자동 배치는 안 한다. 온디맨드로 손으로 돌려본다. 비용은 신규만 파싱.

## 규칙
- 커밋은 작게 쪼갠다. 올리기 전 security-check로 키·개인정보 검사.
- main에 절대 안 올린다. 오직 N186_이재호 브랜치.
- 원문은 재게시하지 않고 상세·신청은 원문 링크로 보낸다.
