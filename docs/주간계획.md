# 2주차 주간 계획 (완료, 보관용)

금요일이 공휴일이라 수·목 이틀로 압축한다. 이번 주 목표는 화면을 만들고(완료), **API 소스 하나로 진짜 데이터를
Supabase에 넣어 화면에 띄우는 것(실데이터 한 바퀴)**까지다. 크롤링과 자유 문장 LLM 파싱은 목 하루엔 무리라
다음 주로 넘긴다. 제일 어려운 크롤링·LLM은 3주차에 제대로 찌른다.

## 요일별

### 월 - 계획·이슈·문서 (완료)
주간 계획, GitHub 이슈, 기획·데이터 전략 정리.

### 화 - React 핵심 화면 (완료)
조건 입력 -> 가능/확인 필요/거의 가능 분류. 검색·필터·즐겨찾기·정렬·상세 모달·컴포넌트 분리.

### 수 - 화면 흐름(라우팅) + 데이터 모델 설계 (완료)
react-router로 홈·공고 상세·즐겨찾기 페이지 분리, 상단 네비. 데이터 모델은 **Supabase에서 프론트가 직접
조회**하는 걸로 결정(Express 조회 서버 안 만듦, 매칭은 클라 match.js). 내일 만들 테이블 SQL(raw_postings,
postings)까지 확정.

### 목 - Supabase 실데이터 한 바퀴 (금요일 공휴일이라 여기서 마무리, 완료)
Supabase 테이블 생성 -> 실데이터 저장 -> 화면이 mock 대신 supabase-js 직접 조회로 교체. 완료.

# 3주차 주간 계획 (진행 중)

## 1. 수직슬라이스 점검 결과 — 이미 완성됨

화면(React) → Supabase 직접 조회 → match.js 판정 → 화면 표시, 이 한 바퀴는 2주차에 이미 완성해서
지금도 실제로 돈다(오늘 헤드리스로 재확인: 실데이터 684건, 판정 결과 가능 306·확인필요 97·거의가능 182,
에러 0). **그래서 이번 주 1순위는 수직슬라이스 완성이 아니라, 이미 도는 것을 총망라·정확도 쪽으로 넓히는 것.**

## 2. 지금까지 실제로 된 것 (주말 로컬 작업 포함)

- **크롤러 5개 소스 완성 + 검증**: 콘테스트코리아, 링커리어(GraphQL), 위비티, 부산청년플랫폼(지자체),
  온통청년(공공API, 지역이 구조 필드로 옴).
- **다중 소스 자동 수집기**(`server/batch/collect.js`): 소스 간 중복 제거(제목 정규화), 노이즈 제외
  (청소년 전용·친목동아리·오디션류), 지자체 지역제한 확인필요 처리, url 기준 dedup upsert.
- **상세화면 즉시 재매칭**(FailEdit): 막힌 조건을 상세에서 바로 고치면 즉시 재판정.
- **디자인 리스킨**: 브랜드 컬러·라운드·그림자·페이지네이션·목데이터 제거·마감일(D-day) 버그 수정.
- **LLM 파서 코드 작성**(`server/parser/extract.js`): 분류+추출 1콜로 합침(비용 절감), 하드 예산 캡
  (30콜/$0.5 - 다른 로직과 독립적인 안전장치), 로컬 비용 추정기. **단, Claude 키가 아직 없어 실제 호출은
  미검증** — 코드만 완성, 검증은 키 오면.
- **GitHub Actions 자동수집 워크플로 작성**(`.github/workflows/collect.yml`) — 로컬에만 있고 아직 push
  안 함(주말 규칙: GitHub 금지).

## 3. 아직 안 된 것

- 콘코 총망라 안전장치: "마감 지난 게 연속 N건 나오면 그 소스 그만 긁기" 룰 — 지금은 페이지 수를 손으로
  정해둔 상태(PAGES 상수). 이게 있어야 비용·시간이 자동으로 실제 규모에 맞춰짐.
- LLM 파싱 실전 검증 — Claude 키 발급 대기 중.
- 한국장학재단 API 연동 — 완료(월1회 스냅샷). 1365 봉사는 맞는 API가 없어 제외. 서울 청년몽땅정보통 추가.
- 정확도 측정(gold 30~50건 라벨링, 정밀도·재현율) — #13/#14.
- GitHub Actions 실제 활성화(시크릿 등록 + push) — 사용자 승인 필요.
- 배포(GitHub Pages) — #16.
