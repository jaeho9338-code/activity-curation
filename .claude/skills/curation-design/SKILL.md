# 대외활동 큐레이션 디자인

이 서비스의 화면은 이 문서의 규칙을 따른다. 새 화면을 만들 때는 이 토큰과 컴포넌트를 그대로 쓰고,
다 만든 뒤에는 아래 체크리스트로 의도에 맞는지 점검한다. 실제 예시는 `design/시안.html`에 있다.

## 방향 한 줄

밝은 보라를 브랜드로, 전부 둥글게, 듀오링고처럼 친근하게. 흰색은 받쳐주는 서브 톤이라 글자가 잘 읽혀야 한다.
상태(가능·불가·확인 필요)는 색으로 한눈에 구분되게 한다.

## 원칙

- 보라가 주인공이다. 브랜드, 버튼, 강조는 보라. 배경은 아주 연한 보라, 카드는 흰색.
- 다 둥글게. 카드는 큰 모서리, 버튼과 칩은 알약 모양. 각지면 안 된다.
- 친근하게. 둥근 캐릭터, 말 거는 듯한 문구("골라줄게", "골라주면 판정할게요").
- 흰색은 서브. 흰 카드에 진한 보라 글자를 올려 잘 읽히게 한다. 흰색을 주인공으로 쓰지 않는다.
- 상태 3색만 예외로 또렷하게. 가능은 초록, 불가는 코랄, 확인 필요는 앰버. 이 셋은 보라에 섞지 말고 분명히 살린다.
- 정보는 촘촘하되 답답하지 않게. 카드 사이 여백을 주고, 근거 문장은 연한 박스에 담는다.

## 색 토큰

새 화면은 이 변수만 쓴다. 임의 색을 새로 만들지 않는다.

```css
:root {
  /* 보라 브랜드 */
  --purple: #7b5cff;
  --purple-hover: #6a47ff;
  --purple-soft: #ede7ff;   /* 연한 보라 패널 */
  --purple-chip: #e7deff;   /* 카테고리 칩 배경 */
  --dark: #241b3d;          /* 다크 퍼플 강조 섹션 */
  --dark-soft: #c9bfe8;     /* 다크 위 보조 글자 */

  /* 표면과 글자 */
  --page: #f3f0ff;          /* 페이지 배경(연보라) */
  --card: #ffffff;          /* 카드(흰색 서브) */
  --border: #ece6fb;
  --ink: #251c40;           /* 기본 글자 */
  --ink-soft: #6e6690;      /* 보조 글자 */
  --ink-muted: #9a93b5;     /* 흐린 글자 */

  /* 상태 3색 (또렷하게) */
  --ok: #17b877;      --ok-bg: #e3f7ef;     /* 지원 가능 */
  --no: #ff6b8a;      --no-bg: #ffe7ec;     /* 불가 / 조건 1개 차이 / 마감 임박 */
  --review: #f5a524;  --review-bg: #fff1d9; /* 확인 필요 */
}
```

## 모서리 / 그림자 / 글꼴

```css
--r-card: 22px;   /* 카드 */
--r-input: 14px;  /* 입력 */
--r-pill: 999px;  /* 버튼, 칩, 필드 */
--shadow: 0 10px 28px rgba(80,50,180,0.12);
--shadow-sm: 0 4px 14px rgba(80,50,180,0.08);
```

글꼴은 둥근 느낌으로. `"Nunito", "Apple SD Gothic Neo", -apple-system, sans-serif`. 제목은 굵게(800).

## 컴포넌트 패턴

- 상단 바: 흰 배경 알약, 왼쪽 브랜드(원형 그라데이션 점 + 이름), 오른쪽 보라 알약 버튼.
- 히어로: 다크 퍼플 카드에 흰 글자, 말 거는 제목, 오른쪽에 둥근 캐릭터.
- 조건 입력: 연보라 패널 위에 알약 필드들(라벨 + 고른 값). 안 고른 건 흐린 글자.
- 결과 섹션: 섹션마다 상태 칩(색 점 + 이름) + 개수. 그 아래 흰 카드 목록.
- 결과 카드: 위에 카테고리 칩과 D-day(임박은 코랄), 굵은 제목, 주최, 연한 근거 박스.
- 조건 1개 차이 카드: 코랄 테두리, "이것만 바꾸면 가능" 배지, 요구값과 내값 비교, 바로 고치기 버튼.

## 점검 체크리스트

디자인을 만든 뒤 이걸로 확인한다. 하나라도 아니오면 고친다.

- 보라가 주인공인가. 흰색이 주인공처럼 넓게 깔리지 않았나.
- 모서리가 다 둥근가. 각진 요소가 없나.
- 가능·불가·확인 필요가 색으로 바로 구분되나. 셋 다 보라에 묻히지 않았나.
- 흰 카드 위 글자가 잘 읽히나. 대비가 충분한가.
- 문구가 딱딱하지 않고 말 거는 느낌인가.
- 위 토큰 밖의 임의 색을 쓰지 않았나.
- 근거 문장이 보이나. 왜 이렇게 판정했는지 사용자가 알 수 있나.

## 마음에 안 들 때 (반복하는 법)

이 스킬을 고정된 규칙으로 두지 말고, 결과가 마음에 안 들면 이 문서를 고친다.
예를 들어 보라가 너무 연하면 `--purple` 값을 바꾸고, 캐릭터를 더 키우고 싶으면 히어로 패턴에 크기를 적어 둔다.
바꾼 규칙으로 화면을 다시 만들어 보고, 될 때까지 문서와 화면을 같이 굴린다.
